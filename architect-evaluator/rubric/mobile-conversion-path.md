# Mobile Conversion Path Rubric

Score whether the mobile architecture gives a user a viable above-fold path to understand, trust, and act.

Axes:

- `mobile_first_viewport_path`: The 390x844 viewport should contain the right opening decision, not only a cropped desktop hero.
- `mobile_action_reachability`: The primary action or its path should be reachable without excessive scrolling.
- `mobile_stack_priority`: Stacked components should preserve the intended hierarchy from Section 5.
- `mobile_friction`: Avoid horizontal overflow, tiny tap targets, hidden required controls, and proof separated from action.

Desktop strength cannot compensate for a weak mobile path. Use the mobile viewport screenshot for above-fold judgment and the mobile full-page screenshot for full sequence only.
