# Visual Design Agent Scaffold

This stage comes after the architecture winner is selected. It should not change product strategy or page structure without a written exception.

Inputs:

- `strategic_diagnosis.json`
- winning `UI_SPEC.{strategy}.md`
- winning `wireframe.{strategy}.html`
- reference move pack
- brand notes, if available

Outputs:

- design system plan JSON
- token recommendations
- high-fidelity visual direction
- implementation notes for the frontend agent
- screenshot acceptance criteria for visual regression

The visual designer should optimize the specific beauty/functionality balance from the diagnosis rather than applying generic SaaS aesthetics.
